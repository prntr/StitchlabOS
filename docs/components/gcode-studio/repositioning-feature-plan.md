# GCode Studio - Canvas Repositioning Feature Plan

## Overview
GCode Studio helps users adapt embroidery designs (from TurtleStitch/Ink/Stitch) to their physical embroidery frame before stitching.

**The Workflow:**
1. **Import**: Load G-Code file generated by TurtleStitch/Ink/Stitch
2. **Set Frame**: Configure physical embroidery hoop/frame size (e.g., 100x100mm)
3. **Position Design**: Move design within frame to optimal position
4. **Export**: Download modified G-Code with updated coordinates
5. **Stitch**: Upload to embroidery machine and stitch at new position

**This feature adds repositioning** so users can move their design within the frame and export the modified G-Code with transformed X/Y coordinates (Z coordinates stay unchanged as they're used for stitch markers, not actual height).

## UI Summary - Where Everything Goes

## Current State Analysis

### Existing Coordinate Systems
1. **Frame Space**: Fixed embroidery frame (0,0 centered), configurable size (e.g., 100x100mm)
2. **Design Space**: G-Code drawing centered around `designCenter`
3. **View Space**: Paper.js viewport with pan/zoom controls

### Current Transformations
- `toPoint(point)`: Converts raw coordinates to Paper.js space (flips Y-axis)
- `toDesignPoint(point)`: Converts to design space centered at `designCenter`
- Design is automatically centered using `designCenter` calculated from bounding box
- Pan/zoom affects entire viewport (frame + design together)

### Existing User Controls
- Mouse drag: Pan entire viewport
- Mouse wheel: Zoom at cursor position
- "Top View" button: Reset view to center
- "Fit to Frame" button: Zoom to fit frame
- Auto-fit on load/resize

## Feature Requirements

### Core Functionality
Allow users to reposition the G-Code design by:
1. **Visual positioning**: Drag/move design on canvas relative to frame
2. **G-Code transformation**: Apply X/Y offset to actual G-Code coordinates
3. **Z-axis preservation**: Keep all Z coordinates unchanged (critical for stitch markers)
4. **Export capability**: Download repositioned G-Code for embroidery machine

**What gets modified:**
- ✅ X coordinates: `X10.5` → `X20.5` (with +10mm offset)
- ✅ Y coordinates: `Y5.0` → `Y0.0` (with -5mm offset)
- 🔒 Z coordinates: `Z1.0` → `Z1.0` (unchanged - used for stitch markers)
- 🔒 Commands: `G0`, `G1`, `G2`, `G3` stay the same
- 🔒 Comments: `;color`, `(STITCH_COUNT:...)` preserved

### Use Cases

**Scenario 1: Center off-center design**
- User exports design from TurtleStitch
- Design loads off-center in 100x100mm frame
- User clicks "Center Design" → design centers in frame
- User exports → embroidery machine stitches centered design

**Scenario 2: Fit design in smaller hoop**
- User has 150x150mm design but only 100x100mm hoop
- User sets frame to 100x100mm
- User repositions design to focus on specific area within hoop bounds
- User exports → only the repositioned portion gets stitched

**Scenario 3: Precise positioning for multi-color designs**
- User needs design at exact position for fabric alignment
- User sets offset to +15mm X, -10mm Y via numeric inputs
- User exports → design stitches at precise location on fabric

**Scenario 4: Avoid hoop edges**
- Design is close to frame boundary
- User repositions away from edges to prevent hoop clips
- User exports → safe stitching with margin from hoop edge

**Scenario 5: Batch workflow**
- User positions multiple designs consistently within same frame
- Export each with same offset for uniform fabric placement
- Upload to embroidery machine for production run

## Proposed Implementation

### 1. Add Design Offset State

```typescript
// New state variables in GCodeStudio2D component
designOffset: { x: number; y: number } = { x: 0, y: 0 }
isDraggingDesign = false
lastDesignDragPoint: paper.Point | null = null
```

Store in Vuex for persistence:
```typescript
// gui.gcodeStudio.designOffsetX
// gui.gcodeStudio.designOffsetY
```

### 2. Update Coordinate Transformation

Modify `toDesignPoint()` to include offset.

### 3. UI Layout Strategy - Three-Tier Approach

- Primary: header toolbar buttons
- Secondary: new Position expansion panel in settings
- Tertiary: canvas overlay feedback

### 4. Interaction Modes

#### Mode 1: Toggle Drag Mode (Simpler)
- Click "Move Design" button to enter drag mode
- Mouse drag now moves design instead of viewport
- Visual indicator (cursor change, highlight)
- Click button again to exit mode

#### Mode 2: Modifier Key (More Flexible)
- Hold Shift while dragging to move design
- Regular drag continues to pan viewport

**Recommendation**: Start with Mode 1 for clarity, add Mode 2 as enhancement

### 5. G-Code Transformation & Export

This is the **critical feature**: actually modifying the G-Code coordinates.

Key rules:
- ✅ X coordinates: add `designOffset.x`
- ✅ Y coordinates: add `designOffset.y`
- 🔒 Z coordinates: never modify
- 🔒 Comments and headers preserved

### 6. Data Persistence

Store offset in `gui.gcodeStudio.designOffsetX/Y`.
